    • Chapter 9: The Window Object

    • Every JavaScript environment when run in a browser has a global window object
    • Refer to global objects without refering to the global object
    • Dialogs: alert() shows text, confirm() shows ok and cancel and returns true or false, prompt() shows text input, ok and cancel and returns text or false
    • The navigator property of the window object returns a reference to a Navigator object which has a userAgent property. This property is unreliable. Don't trust it.
    • window.location
      • window.location.href identifies the current page
      • window.location.protocol returns http, https, pop2, ftp, etc
      • window.location.host returns the domain and port (port is often ommited if 80)
      • window.location.hostname returns just the domain while window.location.port will returns the port or empty string if not explicitly stated in the URL
      • window.location.search returns ? followed by the query string parameters or nothing
      • window.location.hash returns # followed by frametn identifier or nothing
      • window.location.origin returns protocol and domain e.g. https://www.example.com
      • reload() - force reload of current page
      • assign() - load resource from another page
      • replace() - like assign but does not store page in session history so no back navigation
      • toString() - returns entire url e.g. https://www.example.com/somepage
    • window.history
      • window.history.length shows how many pages have been visited
      • window.history.go() takes a number 0 for current page, - to move back and + to move forward e.g. (5) goes forward 5 pages
      • window.history.forward() and window.history.back() can also be use to move pages by 1
    • window.open() will create a new window while window.close will close a window
    • window.moveTo() takes x and y coordinates to relocate a window on the screen
    • window.resizeTo() takes a width and a height
    • All three of the above functions are almost always a bad idea to implement
    • window.screen
      • window.screen.height
      • window.screen.width
      • window.screen.availHeight - height minus operating menus
      • window.screen.availWidth - width minus operating menus
      • window.screen.colorDepth - color bit depth of monitor
    • Every window object contains a document object
    • document.write() outputs a text of string - don't use this, there are much better ways
    • cookies
      • cookies are an early attempt to get around a browsers stateless nature (does not remember anything from one page to the next)
      • cookies are limited to 4kb and can only be read by the domain that set them
      • document.cookie (cookie jar) can be used to show any key/value pair that has been set
      • document.cookie.splie("; ") will turn the stirng into an array
      • Adding expires to a cookie will make it persistent (last beyond the browser session) e.g. document.cookie = `name=Batman; expires=${ expiryDate.toUTCString()}`;
      • path and domain can also be set on a cookie
      • secure will ensure the cookie is transmitted only over HTTPS
      • To delete, set the cookie with a past expiration
    • Timing functions
      • window.setTimeout() accepts a callback and a countdown in ms
      • Using the return value from .setTimeout(), you can stop the callback from happening by passing it to .clearTimeout(...)
      • window.setInterval() is similar to .setTimeout() however it loops the callback according to the countdown
      • clearInterval(...) takes the value returned by .setInterval() to cancel the repeat

    • Chapter 15: HTML5 APIs

    • To embed data in a webpage, use the data- attribute
    • Multiple data- attributes can be accessed via the 'dataset' property of an element e.g. element.dataset.powers where powers equals data-powers or element.getAttribute('data-powers')
    • localStorage is used to level-up cookie usage with .setItem(key, value) and .getItem(key)
    • removeItem(key) is used to remove an entry from localStorage or clear() to remove everything
    • Get around the string only nature of localStorage using JSON.stringify() and JSON.parse()
    • navigator.geolocation.getCurrentPosition(callback) sends a position object with a coords property containing latitude and logitude to the specified function
    • position.speed returns groundspeed
    • position.altitude returns device estimate in meters
    • position.heading returns direction
    • position.timestamp returns when the information was recorded
    • position.accuracy to indicate the truthiness of the latitude and longitude (lower is more accurate)
    • position.altituteAccuracy is similar
    • geolocation.watchPosition(callback) will call a callback every time the position is updated
    • clearWatch with return of watchPosition to cancel
    • Websockts are a new protocol that allows two-way communication
    • Notificaitons allow you to show messages using the system notifications
    • Use <audio> and <video> to display multimedia
    • Other HTML5 APIs

    • Shims and Polyfills

    • While a shim adds missing browser functionality, a polyfill uses the native API if it exists and if not uses added code
