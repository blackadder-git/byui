  • Notes

    • Objects

      • Representations of real-world entities
      • A method added as a function expression to a user object:
        user.sayHello() = function() {
        alert("Hello");
        };
      • The same method could also be added directly to the class definition e.g. sayHello : function() {
        alert("Hello");
        }
      • The following shorthand will also work: sayHello() {
        alert("Hello");
        }
      • When used, 'this' represents the current object
      • Arrow functions do not have this, instead if used, this references the outer context
      • A function without parentheses will return the function definition

    • this

      • "this" is a keyword whose value changes depending on how a function gets called
        • global context - this defaults to the window object when called outside of a function
        • object construction - refers to the instance of the new object
        • object method - refers to the instance of the object in which it is called (similar to this in a constructor)
        • simple and anonymous function (even when called from an object) - when not associated with an object, this in a function refers to the Window object
        • arrow function - this in arrow is the same as the immediate outside scope
        • event listener - in an event, this is set to the element that fired the event

    • bind

      • Every function has this method. It is used to change the this context. The first parameter passed to bind becomes the new this.
      • bind doesn't work with arrow functions

    • Object literals

      • An object created by wrapping properties and methods within curly braces
      • Properties are key-value pairs separated by colons
      • Methods include parentheses and curly braces
      • Access properties with a dot or bracket notation
      • Using bracket notation has the advantage of letting you create dynamic property names
      • Use 'in' to check for the existence of an object property =R use !== to see if the property is undefined
      • While the two options above work for all properties, .hasOwnProperty()= will return true only if the object has a matching property which was not inherited
      • Use for(const key of Object.keys(...)) or a for(const key in ...) to list out the names of an objects properties
      • Use for(const value of Object.values(...)) to get values
      • Use for(const [key, value] of Object.entries(...)) to get both keys and values
      • Properties and methods can be added to an object via assignment = e.g. superman.fly = true
      • Properties and methods can be deleted from an object via delete e.g. delete superman.fly
      • When an object is copied by assignment, both objects point to the same space in memory
      • Passing parameters to a function as an object in curly braces is refered to as 'named parameters'
      • Namespacing involves wrapping properties and functions with an object
      • JSON is an object literal represented as a string
        • Property names need to be double-quoted
        • Allowed values are quoted strings, numbers, boolean, null, arrays and objects
        • No functions, if present, they are ignored
        • parse() takes a JSON string and returns a JavaScript object
        • stringify() takes a JavaScript object and returns a string of JSON data
      • Math is a built in object
        • Includes build in constants like Math.PI, Math.E, Math.LN2, etc
        • Methods include abs(), ceil() up to next integer, floor() down to next integer, round() to nearest int, trunc() return int, lose decimal, exp(), pow(), sqrt(), cbrt(), hypot(), log(), log2(), log10(), max(), min(), sin(), cos(), tan(), asin(), acos(), atan(), sinh(), cosh(), tanh()
        • JavaScript uses random() to generate random numbers
      • Date represents a single moment in time
        • Without an arguement, new Date() will return the current date and time
        • Although a variety of strings can be used, best practice is to use separate arguments e.g. new Date(year,month,day,hour,minutes,seconds,milliseconds);
        • Timestamps are also valid (milliseconds since Epoch 1st Jan 1970)
        • getters and setters are used to access a date object's data
      • RegExp
        • Once a regular expression is created, pass a string to the test method to see if it is false or true e.g. pattern.test('applesauce');
        • pattern.exec('applesauce') is similar but returns null or an array with info
        • Can be just a string of characters
        • Groups of characters are placed inside brackets []
        • A sequence of characters is represented by placing a - between the first and last character A-Z or 0-9
        • ^ before the sequence negates it
        • Complex patters can be made by enclosing different groups in brackets [] e.g. [Gg]o[Ll]d matches Gold, goLd, GoLd, gold
        • Regular expressions have the following properties: global, ignoreCase, multiline. TO use, place g, i, m after the expression to set true e.g. /java/i
        • Metacharacters
          • . matches any character but line break
          • \w matchs any word, same as [A-Za-z0-9_]
          • \W matches any non-word, same as [^A-Za-z0-9_]
          • \d matches any digit, same as [0-9]
          • \D matches any non-digit, same as [^0-9]
          • \s matches whitespace, same as [\t\r\n\f]
          • \S matches non-whitespace, same as [^\t\r\n\f]
        • Modifiers
          • ? preceding token is optional
          • * matches zero or more occurrence of the preceding token
          • + matches one or more occurence of the preceding token
          • {n} matches n occurences of the preceding token
          • {n,} matches at least n occurences of the pattern
          • {,m} matches at most m occurrences of the preceding token
          • {n,m} matches at least n but not more than m occurrences of the preceding token
          • ^ marks the position immediately before the first character in the string
          • $ marks the position immediately after the last character in the string
          • Escape special characters and modifiers with a backslash
          • To make a modifer lazy, make a greedy value optional with ?
          • split(), match(), search(), replace() all accept regular expressions

    • Document Object Model

    • A node representation of an HTML page
    • Calling .nodeType on a node will return: element, attribute, text, comment, body
    • getElementById(), getElementByTagName(), getElementsByClassName()
    • document.querySelector(), documnet.querySelectorAll()
    • The above methods can be called on any element, not just document
    • jQuery uses these same methods when it can
    • childNodes returns all the nodes of a parent element (including text and whitespace nodes)
    • children returns only element nodes (no text or whitespace)
    • firstChild and lastChild do what they suggest
    • parentNode, nextSibling, previousSibling,
    • textContent and innerText (IE 8) return the text content of a node
    • getAttribute and setAttribute do what you expect
    • Use classList.add(...) or classList.remove(...) rather than className to modify an elements class. classList.toggle(...) also works and classList.contains(...) is also useful
    • document.createElement(...) will return a new item as a DOM fragment
    • appendChild(...) is used to add fragments to the page
    • removeChild(...) (called on the parent object), replaceChild(...)
    • innerHTML respects html tags while textContent treats tags as text
    • innerHTML has the advantage of automatically creating element nodes (sans code within script tags) as part of the text
    • Every element has a style property
    • CSS styles which are separated by dash uses camelCase notation in JavaScript. If bracket notation is used e.g. style['background color'] camelCase is not used

    • Events

    • JavaScript event listening is non-blocking
    • Rather than inline events onclick="..." use document.body.addEventListener("click", function); (event type, action to take)
    • When an event listener lacks a node, it is global to the entire page
    • The Event Object
      • type - e.g. click
      • target - node that fired the event
      • screenX, screenY - number of pixels from left and top in the screen
      • pageX, pageY - number of pixels from left and top where the event took place in the document
    • Events
      • click
      • dblclick
      • mousedown
      • mouseup
      • mouseover
      • mouseout
      • keydown - key press
      • keyup - key release
      • keypress - character typed on the screen
      • touchstart - better not to use as event may be the start of a zoom or swipe
      • touchend - user stops touching the screen
      • touchmove - touches the screen
      • touchenter - touching surface then passes over element
      • touchleave - still touching the surface but leaves the element connected to the event
      • touchcancel - too many fingers, pop-up dialog
    • keyboard events can use event.key === 'c' to test the key
    • Touch Event Properties
      • length = number of fingers
      • screenX
      • screenY
      • radiusX
      • radiusY
      • identifier
    • removeEventListenter('click', function)
    • preventDefault() keeps an object from performing its default behavior
    • Event propagation
    • Events when clicked bubble (up) (default behavior) or capture (down)
    • event.stopPropagation() can be used in a listener event to keep an event from propagating ... but why?
    • Target the parent of an item and then target multiple items at once vs creating multiple event handlers
