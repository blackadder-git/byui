    • Chapter 2: Programming Basics

      • Comments

      • Use // for single line, /* */ for multi-line
      • Generate doc with JSDoc, Tooklit, Docco, YUIDoc

      • Grammar

      • Best practice - separate each statement with a new line, terminate each with a semi-colon
      • Reserved words - abstract, await, boolean, break, byte, case, catch, char, class, const, continue, debugger, default, delete, do, double, else, enum, export, extends, false, final, finally, float, for, function, goto, if, implements, import, in instanceof, int, interface, let, long, native, new, null, package, private, protected, public, return, short, static, super, switch, synchronized, this, throw, throws, transient, true, try, typeof, var, volatile, void, while, with, yield (not reserved but should be = undefined, NaN, Infinity)

      • Primative Data Types

      • String, Symbol, Number, Boolean, Undefined, Null (anything else = object e.g. array, function, object literal)
      • Access properties of a primative type with dot notation (string.length) or bracket notation (string['length'])

      • Variables

      • const (fixed variables) vs let (fluid variable, may be reassigned later)
      • If const is an object, data inside can change
      • Best practice - use const whenever you can
      • Separate variables with a comma to assign multiple variables at the same time

      • Scope

      • const and let have block scope; their values only exist inside the block in which they are declared
      • global (everywhere) vs local (block)
      • Not using let or const means the variable will have global scope

      • Naming

      • Case sensative, may begin with any letter, underscore (convention used for private) or $ (used by jQuery), may include numbers but can not start with them
      • camelCase vs PascalCase vs snake_case

      • Assignment

      • Primative types assign values directly while non-primative types assign values by reference
      • let a = 2; let b = a; b = 100; a = 2, b = 100
      • let a = {"honey"}; let b = a; b = {"bee"}; a = bee, b = bee

      • Strings

      • Common methods: toUpperCase(), toLowerCase(), charAt(), indexOf(), lastIndexOf(), includes(), startsWith(), endsWith(), concat(), repeat()

      • Template Literals

      • Begin and end with `, use ${} to interpolate (insert JavaScript code)

      • Symbols

      • Symbols are new primative data types created with Symbol(<description>)

      • Numbers

      • Numbers = integers (whole numbers) or floats
      • Common methods: toFixed(), toPrecision()
      • Built-in methods accessed with: .. e.g. 5..toExponential(), space . e.g. 5 .toExponential(), decimal form 5.0.toExponential(), parentheses (5).toExponential()
      • JavaScript supports hex (0x...), octals (0o...) and binary (0b...)

      • Exponentials (multiply by 10 to the power of)

      • e.g. 1e6, 2E3, 10**3

      • Arithmetic Operations

      • add +, subtract -, multiply *, divide /, exponent **, ã€œmodulus % (result always has the same sign s the first number)

      • Infinity

      • Represents any number that is too big for the language to handle
      • The biggist number in JavaScript is 1.7976931348623157e+308
      • The smallest number (-Infinity) is -1.7976931348623157e+308

      • NaN

      • This error results from operanding something that is not a number e.g. "Hello" * "Matey"
      • Use Number.isFinite(...) to verify the result is not Infinity, -Infinity, NaN

      • Convert Strings to Numbers or Numbers to Strings

      • Best practice is to use Number(...) or String(...)
      • Alternatively, for numbers multipy a numeric string by 1 e.g. '5' * 1, put an operator before the string e.g. +'5'
      • And for strings add empty e.g. 3 + '', or toString() e.g. 10..toString(), or parseInt('...', base)

      • Undefined and Null

      • Undefined = missing value, while Null = no value

      • Booleans

      • true and false (named after George Boole)
      • Always false = "", '', ``, 0, -0, Nan, false, null, undefined

      • Logical Operators

      • ! (NOT) before a value returns the opposite, !! returns false or true shorthand for Boolean(...)
      • && (AND) evaluates to true when all operands are true
      • || (OR )evaluates to true when one or more operands are true

      • Bitwise Operators

      • ~ (NOT) flips all the bits in a 32-bit integer
      • & (AND) combines two numbers and keeps all the 1's where both operands are 1
      • | (OR) combines two numbers and keeps all the 1's where either operands is 1
      • ^ (XOR) combines two numbers and keeps all the 1's where either operands is 1 but not both

      • Bitwise Shift Operators

      • << multiply by power of 2
      • >> divides by power of 2
      • Use 2 when right hand number = 1, otherwise raise 2 by the right hand number then multiply or divide e.g. 5 << 3 = (5 * 2**3) = 40

      • Comparison

      • == (soft equality), === (hard equality) checks type too, likewise, != and !==
      • No hard and soft less than, greater than operators, instead use < and > || with === e.g. 8 > '8' || 8 === '8'
      • NaN === Nan is false, use Number.isNan(Nan) instead
      • Strings can also be tested with upper-case letters < lower-case letters
    • Chapter 3: Arrays, Logic and Loops

      • Arrays

      • Best practice is to create an array literal with []
      • Values can be destructured from an array as follows: const [x, y] = [1, 2]
      • Use indexOf() to search for a specific value or includes() which will return a boolean value
      • JavaScript supports multidimensional arrays, use two indexes to access values
      • ... spread operator will flatten the values in a multidimensional array
      • Summary on Sitepoint

      • Sets

      • Sets are collections of unique data
      • Create with new Set();
      • Add with .add() or using [] in the constructor
      • A set can contain duplicate values only if they are in the form of an array or object
      • Common methods: size(), delete(), clear()
      • If adding non-primatives to a set, use WeakSet() otherwise, memory leaks can occur as JavaScript may not be able to garbage collect elements inside of the set
      • WeakSets don't know what they contain, use .has() to check for specifics

      • Maps

      • Key/value pairs, maps are similar to hashes or dictionaries
      • While objects must use strings for keys, maps can use any data type
      • Create with new Map();
      • Add with set(<key>,<value>);, Map also accepts a nested array e.g. [ [<key>,<value>], [<key>,<value>] ]
      • To retrieve data, use get(<key>), or has() which will return a boolean value
      • delete() will remove a single item while clear() will remove all key/values
      • WeakMaps work just like WeakSets, no primative values and auto garbage collection

      • Logic

      • JavaScript supports: if/else, (condition) ? (do if true) : (do if false), switch, while, do (while), for
    • Chapter 4: Functions

      • Function Declarations

      • function hello() {
        ...
        }
      • Automatically hoisted

      • Expressions (literal, unnamed, anonymous)

      • const hello = function() {
        ...
        };
      • Invoke a function which is assigned to a variable by adding parentheses e.g. hello();

      • Named Expressions

      • const hello = function greeting() {
        ...
        };

      • Arrow Functions

      • (x) => {
        ...
        };
      • () => {
        ...
        };
      • parameters appear between = signs
      • Function parameters are defined in the function definition, function arguements are sent to the function when it is invoked
      • Any arguement not provided to a function when it is invoked is set to undefined, extra arguments are ignored
      • Because functions can accept any number of arguments, to collect them for review, place the rest operator ... before the last argument in the fuction
      • Functions accept default parameters as hello(arg="world"), default parameters should come last
      • Callbacks are functions passed as parameters to other functions
