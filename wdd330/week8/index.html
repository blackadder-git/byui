    • Chapter 8: CSS Transforms and Transitions

    • transform lets you translate, rotate, scale or skew any element on the page
    • transforms can happen in 2D or 3D space
    • Rather than declare multiple transformations, use a space separated list e.g. transform: rotate(10deg) translateX(40px) scale(1.5);

    • Transforms

    • Translate
      • Move elements left, right, up, down
      • translateX or translateY if you only want to move along one axis
      • Transforms donÂ´t work on inline elements, to fix redefine them as display: inline-block
      • Transforms require vendor prefix to work in IE9, Android and Blackberry
      • transform: translate(45px,-45px);
    • Scaling
      • Using one value increases or decreases the element while two different values distorts the element
      • Scaled elements do so from the center point unless you use the transform-origin property to change the default behavior
      • transform: scale(1.5, 0.25);
    • Rotation
      • Rotate an element around the point of origin, default center
      • transform: rotate(10deg);
    • Skew
      • First element skews on x, second on y
      • transform: skew(15deg, 4deg);

    • Transitions

    • Transitions allow the values of CSS properties to change over time
    • Steps to recreate
      • Declare original state of element in default style declaration
      • Declare final state of element in a transitioned element e.g. :hover
      • Include transition functions in default style declaration
    • transition-property
      • Separate properties by comma
      • transition-property: transform;
        transition-duration: 0.2s;
        transition-timing-function: ease-out;
        transition-delay: 50ms;
      • Can be rewritten as this: transition: transform 0.2s ease-out 50ms;
      • To specify different transitions to different properties, include them in a comma separated list as follows
      • transition-property: transform, color;
        transition-duration: 0.2s, 0.1s;
        transition-timing-function: ease-out, linear;
        transition-delay: 50ms;
    • transition-duration
      • Determines how long the transition lasts, use secons (s) or milliseconds (ms)
      • 200ms is considered optimal transition
    • transition-timing-function
      • Determines pace of transition (slow -> fast, fast -> slow, even keel)
      • ease - slow start, speeds up, slows down
      • ease-in-out - sharper start acceleration
      • linear - constant transition
      • ease-in - slow start, speeds up, stop
      • ease-out - fast start, slows down
      • Create your own transitions
    • transition-delay
      • Time to wait for the transition to begin, 50ms is normally good

    • Animations

    • Keyframe
    • A keyframe is a snapshot that defines the start or end point of any smooth transition
    • To use, first create a named animation, then attach it to an element in its property declaration block e.g. @keyframes { ... }
    • Each keyframe block uses a percentage value with two keyterms from and to which are between 0% and 100% to indicate where they fall along the animation
    • @keyframes moveRight {
      from {
      transform: translateX(-50%);
      }
      to {
      transform: translateX(50%);
      }
      }
    • At a minimum, an element needs an animation-name: and an animation-duration:
    • Other animation properties
      • animation-timing-function (similar to transition-timing-function) can also be used
      • animation-iteration-count - how many times to play the animation e.g. 6 or infiniate
      • animation-direction - normal, reverse, alternate, alternate-reverse (change direction)
      • animation-delay
      • animation-fill-mode - none, forwards, backwards, both
      • animation-play-state
    • animation-name: appearDisappear;
      animation-duration: 300ms;
      animation-timing-function: ease-in;
      animation-iteration-count: 1;
      animation-direction: alternate;
      animation-delay: 5s;
      animation-fill-mode: backwards;
      animation-play-state: running;
    • Shorthand: animation: 300ms ease-in alternate 5s backwards appearDisappear;

    • Chapter 12: Canvas, SVG, and Drag and Drop

    • Canvas

    • When drawing on a canvas context object, the top left corner is 0,0. The system reads top to bottom, left to right
    • Gradients and images can also be loaded onto the canvas context
    • https://www.youtube.com/watch?v=09fisjQOYtk
    • Canvas lacks accessibility. There is no DOM note, is not text based and is therefore 'invisible' to screen readers
    • Drawings can be saved

    • SVG

    • A Scalable Vector Graphic uses XML to describe its points thus maintaining quality as it is scaled up or down
    • SVG images are available via the DOM
    • Unlike Canvas elements, SVG is perserved in the XML document that describes it

    • Drag and Drop

    • Unsupported by Android and by design iOS
    • Used to drag files from computer to web page or for dragging elements to other elements
      • Set draggable attribute of any html element e.g. draggable="true"
      • Add event listener for dragstart event
      • Add event listener for dragover and drop event to any area where an item can be placed
